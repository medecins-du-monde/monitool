Welcome to Monitool!

Monitool is an addon of the intranet

It aims to centralize the data generated by the NGO's field projects

Why

  • Increase reliability
    • Reduce the learning curve required to consult data
    • Have consistency checks on data to limit entry errors
    • Limit mistakes in calculation
  • Stepping back
    • Encourage geographically remote projects to use the same indicators when measuring comparable events.
    • Have mechanisms to compare the value of an indicator on a project, with the same value for all other that collected it.
  • Ease communication
    • Provide direct access to project's data for people outside of the operational services of the NGO (management, communication, ...)
    • Have single source of truth between headquarters and field project

When

Monitool plays a role during the four stages of a project

  • Design of the logical frame
    Monitool provides all others projets logframes, with the list of used indicators and targets for each one of them.
  • Set up of monitoring tools
    Explicitely write down the path followed by your data and attach deadlines on data collection.
  • Follow-up
    Reporting graphs and tables of your choice are available and built with the most up-to-date data
  • Post-closing
    Your projects are archived and available for future reference!

Before starting

  • Make a list of all reports that you will need to create on the project's life
  • List all indicators that are contained in those reports, and other indicators you need for monitoring
  • List all data that you will need to extract to compute those
  • Indentify which data source will provide with this data

Bienvenue sur Monitool!

Monitool est une extension de l’intranet de l’ONG

Il a pour objectif de centraliser les données générées par les projets de Médecins du Monde.

Pourquoi

  • Augmenter la fiabilité
    • Réduire la courbe d’apprentissage nécessaire à l'exploitation des données.
    • Disposer de contrôles de cohérence afin de limiter les erreurs de saisie
    • Limiter les fautes de calcul
  • Prendre du recul
    • Encourager des projets géographiquement éloignés à utiliser les mêmes indicateurs lorsqu’ils mesurent des grandeurs comparables.
    • Disposer de mécanismes pour comparer la valeur d’un indicateur sur un projet, avec la même grandeur chez tous les autres qui le collectent.
  • Faciliter la communication
    • Donner accès direct aux données des projets aux services hors opérationnel de l’ONG (direction, communication, ...)
    • Disposer d'une unique version des données partagée entre siège et terrain

Quand

Monitool joue un rôle pendant quatre étapes d'un projet

  • Dans la conception du cadre logique
    Avec la mise à disposition des cadres logiques des autres projets, et du catalogue des indicateurs, accompagné des bases, cibles et collectes passées pour chacun d'eux.
  • Mise en place des outils de suivi
    Explicitez le chemin suivi par vos données et fixez des échéances sur leur collecte
  • Suivi régulier
    Les graphes et tableaux de reporting de votre choix sont disponibles et toujours réalisées avec les données les plus à jour disponible
  • Post-fermeture
    Vos projets restent archivés et disponibles pour toute consultation future!

Avant de commencer

  • Faites une liste de tous les rapports que vous devrez créer pendant la vie du projet
  • Listez tous les indicateurs qui sont contenus dans ces rapports
  • Listez toutes les données à collecter sur le project pour les calculer
  • Identifiez des sources de données pour chacune de ces données

Bienvenidos a Monitool!

Monitool es una extensión de la intranet de la ONG

El objetivo del proyecto es centralizar los datos generados por los proyectos de Médecins du Monde

¿Por qué

  • Aumentar la fiabilidad
    • Reducir la curva de aprendizaje necesaria para el analisis de datos
    • Hacer pruebas de coherencia para limitar los errores de entrada
    • Limitar los errores de formulas en indicadores
  • Tomar distancia con los datos
    • Fomentar proyectos geográficamente remotos a utilizar los mismos indicadores cuando miden magnitudes comparables.
    • Contar con mecanismos para comparar el valor de un indicador en un proyecto, con el mismo valor para todos los demás.
  • Facilitar la comunicación
    • Proporcionar acceso a los datos de los proyectos a servicios que estaban excluídos (gestión, comunicación, .. ).
    • Tener una versión única de los datos compartidos entre la sede y los terrenos

Cuando

Monitool tiene un papel durante cuatro etapas de un proyecto

  • Diseño del marco lógico
    Con la provisión de los marcos lógicos de los demas proyectos, sus indicadores, valores de base y objectivos.
  • Creación de las herramientas de supervisión
    Ayuda a explicitar el camino seguido por los datos y a fijar plazos límite
  • Seguimiento
    Cree gráficos y tablas con los datos mas la dia disponibles
  • Post-cierre
    Sus proyectos son archivados y disponibles para referencias futuras!

Para empezar

  • Cree una lista de todos los informes que se deberan crear durante la vida del proyecto
  • Cree la lista de los indicadores que estos informes contienen
  • Cree la lista de los datos que deberan ser colectados en el proyecto para calcularlos
  • Identifique fuentes de datos para cada dato
